# Projekt-Checkliste

## Aufgaben aus dem Team-Meeting

- [ ] GitHub-Repository fuer das Projekt anlegen
- [x] Frontend und Backend sauber getrennt
- [x] Requirements-Dateien erstellt (`frontend/requirements.txt`, `backend/requirements.txt`)
- [ ] Teamregel: taeglich mindestens 1 Commit pro Person
- [ ] Gemeinsame Online-Zeit vormittags (09:00 bis 12:00 Uhr) einhalten
- [ ] Bewertungskriterien von Mete einarbeiten (sobald verfuegbar)

## Technische Pflichtkriterien

- [x] Datenbankanbindung (SQLite)
- [x] Login-System
- [x] CRUD fuer Hauptressource (Rezepte)
- [x] CRUD fuer Nebentabellen (Kommentare)
- [x] Python-Skript fuer API-Interaktion
- [x] README mit Projektbeschreibung, API-Uebersicht und Daily-Dokumentation

## Naechste Team-Schritte

- [ ] Repository auf GitHub pushen
- [ ] Teammitglieder in README eintragen
- [ ] Taegliche Commit-Routine starten
- [ ] Abschlussdemo vorbereiten
- [ ] Anleitung `ANLEITUNG_WIE_WIR_VORGEHEN.md` gemeinsam durchgehen
